<objective>
Implement the PPU with 4-mode state machine, background rendering with scroll, window layer, STAT/LYC interrupt, and VBlank interrupt. The PPU fills a 160x144 framebuffer on each frame.

**Output**: Working PPU rendering BG + Window via a cycle-accurate mode state machine.
</objective>

<task type="auto">
  <name>Task 1: Create PPU struct and state machine</name>
  <files>
    src/ppu.zig (new)
    src/addr.zig
  </files>
  <action>
    Create `src/ppu.zig` with:

    **PPU struct fields**:
    - `lcdc: u8` — LCD control (mirror of MMIO)
    - `stat: u8` — STAT register (mode flags + LYC=LY + interrupt enables)
    - `scy, scx: u8` — scroll Y/X
    - `ly: u8` — current scan line (0-153)
    - `lyc: u8` — LY compare
    - `wy, wx: u8` — window position
    - `bgp: u8` — background palette
    - `dot_counter: u16` — dots elapsed in current line (0-455)
    - `mode: PpuMode` — current PPU mode enum { OAMScan, Drawing, HBlank, VBlank }
    - `framebuffer: [160*144]u8` — 2bpp output (2 bits per pixel, 4 shades)
    - `vram: [8192]u8` — VRAM (tile data + tile maps)
    - `oam: [160]u8` — OAM (sprite attribute table) — 40 entries × 4 bytes
    - `bus: *Bus` — back-reference for interrupt requests

    **PpuMode enum** (u2):
    - `oam_scan = 2` (mode 2)
    - `drawing = 3` (mode 3)
    - `hblank = 0` (mode 0)
    - `vblank = 1` (mode 1)

    **Methods**:
    - `init()` — initialize fields, zero framebuffer
    - `tick(mcycles: u4)` — advance PPU by N M-cycles
    - `getFramebuffer()` — return framebuffer slice

    **tick()** should:
    1. Convert M-cycles to dots: `dots = mcycles * 4`
    2. Call `stepDots(dots)` N times

    **stepDots(dots: u16)**:
    - Add dots to dot_counter
    - If LY >= 144 and mode != VBlank: transition to VBlank, set IF bit 0, set STAT bit 1
    - Switch on current mode:
      - OAMScan: if dot_counter >= 80, transition to Drawing
      - Drawing: if dot_counter >= 80+172 (252), transition to HBlank
      - HBlank: if dot_counter >= 456, increment LY, if LY < 144 reset to OAMScan, else stay in VBlank
      - VBlank: if dot_counter >= 456, increment LY, if LY >= 154 reset LY to 0 and reset to OAMScan
    - Update STAT register mode bits on each transition
    - Handle LYC=LY: set STAT bit 2, fire STAT interrupt if enabled

    **State transitions**:
    - Mode 2→3: set STAT mode bits to 3
    - Mode 3→0: set STAT mode bits to 0, fire STAT mode-0 interrupt if enabled
    - Mode 0→2 (next line): set STAT mode bits to 2, fire STAT mode-2 interrupt if enabled
    - Mode 1→2 (frame reset): reset LY to 0, set STAT mode bits to 2

    **VRAM/OAM** — for now these are separate buffers in PPU struct, not yet wired to bus.
  </action>

    <automated>zig build 2>&1</automated>
  </verify>
  <done>
    PPU struct with mode state machine created. tick() advances dots and transitions between modes 2/3/0/1. VBlank interrupt fires on LY=144.
  </done>

<task type="auto">
  <name>Task 3: Background rendering with scroll</name>
  <files>
    src/ppu.zig
    src/addr.zig
  </files>
  <action>
    Implement background rendering in `ppu.zig`:

    **During mode 3 (Drawing)**:
    - For each dot in the visible line (dots 80-251, 0-159 pixels):
      - Calculate pixel X: `pixel_x = (dot - 80) + scx % 8` (tile-aligned)
      - Calculate tile map address: based on LCDC bit 3 (0x9800 or 0x9C00)
      - Calculate tile Y: `(ly + scy) / 8` (tile row)
      - Calculate tile X: `pixel_x / 8` (tile column)
      - Read tile index from tile map
      - Read tile data: tile index * 16 bytes (2 planes × 8 rows)
      - Calculate pixel within tile: `((ly + scy) % 8) * 2` for tile row
      - Extract color ID (2 bits) from tile data planes
      - Map color ID through BGP palette to shade value
      - Write shade to framebuffer[ly * 160 + visible_x]

    **Tile data formats**:
    - LCDC bit 4 = 1: unsigned addressing (0x8000 base, tile 0-255)
    - LCDC bit 4 = 0: signed addressing (0x8800 base, tile 128-255 = -128 to -1, 0-127 = 0 to 127)
    - Each tile is 16 bytes: 2 planes of 8 bytes each
    - Plane 0: bytes 0, 2, 4, ..., 14 (even bytes)
    - Plane 1: bytes 1, 3, 5, ..., 15 (odd bytes)
    - Color ID = (plane1_bit << 1) | plane0_bit

    **Scroll**:
    - SCY adds to LY for tile Y calculation
    - SCX shifts pixel X
    - SCX % 8 pixel offset: the first visible pixel might be partway through a tile

    **Performance note**: For now, render the full background line during mode 3 entry (not per-dot). This is simpler and still functionally correct.
  </action>

<task type="auto">
  <name>Task 4: Window layer rendering</name>
  <files>
    src/ppu.zig
  </files>
  <action>
    Implement window layer overlay on the background:

    **Window behavior**:
    - Window is enabled when LCDC bit 5 = 1
    - Window overrides background pixels in its area
    - Window position: WX (x-7, since DMG convention has WX+7 as actual x), WY
    - Window tiles come from tile map selected by LCDC bit 6 (0x9800 or 0x9C00)
    - Window uses the same tile data as BG (LCDC bit 4)
    - Window has its own internal Y counter (wy_counter) that tracks which window line to render

    **Window rendering**:
    - After background is rendered on a given line, if window is enabled and LY >= WY:
      - Calculate window pixel X: pixel_x - (wx - 7)
      - Calculate window internal Y: wy_counter (increments each time a window line starts)
      - Use the window tile map to get tile indices
      - Read tile data same as BG
      - Write pixel to framebuffer (overwrites BG pixel)

    **Window Y counter**:
    - Incremented each time a window scan line is rendered
    - Reset to 0 when LY goes from 153→0 (start of new frame)
    - Only increments when LY >= WY and window is enabled
  </action>
