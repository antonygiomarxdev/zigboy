  <action>
    Create `tests/blargg.zig`:

    **ROM URL** (D-06):
    - Define `rom_url = "https://raw.githubusercontent.com/retrio/gb-test-roms/master/cpu_instrs/cpu_instrs.gb"`
    - Define `rom_cache_path = "tests/roms/cpu_instrs.gb"` (relative to workspace root, D-07).

    **Fetch implementation** (D-05, D-08, agent's discretion: try std.http first):
    - `fetchRom(allocator: std.mem.Allocator, url: []const u8, output_path: []const u8) !void`:
      - Open output file for write via `std.fs.cwd().createFile(output_path, .{})`.
      - Try `std.http.Client` first:
        - Create client: `var client = std.http.Client{ .allocator = allocator }; defer client.deinit();`
        - Parse URI via `std.Uri.parse(url)`.
        - Create body ArrayList for response storage.
        - Call `client.fetch(.{ .method = .GET, .location = .{ .uri = uri }, .response_storage = .{ .dynamic = &body } })`.
        - Check response status `.ok`. Write body to output file.
      - Fallback to curl if std.http fails:
        - Run `std.process.Child.run(.{ .allocator = allocator, .argv = &[_][]const u8{ "curl", "--silent", "--fail", "--output", output_path, url } })`.
        - Check exit code == 0.
      - If both fail: return `error.TestRomFetchFailed`.
    - `ensureRomExists(allocator: std.mem.Allocator) ![]const u8`:
      - Check if `tests/roms/cpu_instrs.gb` exists (try `std.fs.cwd().access(rom_cache_path, .{})`).
      - If missing: try to create `tests/roms/` directory via `std.fs.cwd().makePath("tests/roms")`.
      - Call `fetchRom(allocator, rom_url, rom_cache_path)`.
      - If fetch fails: `std.log.err("Failed to download test ROM. Check network or manually place at {s}", .{rom_cache_path}); return error.TestRomFetchFailed;`
      - Return the path to the cached ROM.

    **Test runner**:
    - `test "Blargg cpu_instrs"`:
      - Allocator: create `std.testing.allocator` (standard Zig test allocator).
      - `const rom_path = try ensureRomExists(std.testing.allocator);`
      - Read ROM file: `const rom_bytes = try std.fs.cwd().readFileAlloc(std.testing.allocator, rom_path, 8 * 1024 * 1024); defer std.testing.allocator.free(rom_bytes);`
      - `var emu = try Emulator.init(std.testing.allocator, rom_bytes); defer emu.deinit();`
      - Run emulator for up to ~60 million T-cycles (≈ 55 emulated seconds, generous upper bound):
        - Use `runForFrames(60 * 60)` (60 FPS × 60 seconds = 3600 virtual frames).
        - Alternatively, run in a loop checking `emu.getSerialOutput()` for "Passed" every ~1000 frames, breaking early when found.
      - After loop: `const output = emu.getSerialOutput();`
      - `try std.testing.expect(std.mem.indexOf(u8, output, "Passed") != null);`
      - If this fails, print the serial output content (or first 200 chars thereof) to help debugging: `std.debug.print("Serial output:\n{s}\n", .{output[0..@min(output.len, 200)]});`

    **Environment variable override** (D-26):
    - Check `std.process.getEnvVarOwned(std.testing.allocator, "ZIGBOY_TEST_ROM")` — if set, use that path instead of auto-fetch. This allows CI to point at a pre-cached ROM.

    **Timeout consideration** (RESEARCH.md A1):
    - cpu_instrs.gb takes ~55 emulated seconds on real DMG. At 1x emulation speed, the test may take ~1-5 wall-clock seconds on a modern CPU (emulation is faster than real hardware). Use a generous frame limit of 3600 frames (60 seconds at 60 FPS).
